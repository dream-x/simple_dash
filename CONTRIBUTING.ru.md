# Участие в Simple Dash

Спасибо, что решили внести вклад. Simple Dash — это небольшой homelab-инструмент без зависимостей, и цель — таким его и сохранить: легко читать, легко запускать и безопасно направлять на домашнюю сеть.

---

## Базовые правила

- **Никаких сторонних зависимостей.** Генератор и сервер используют **только стандартную библиотеку** Python. Если изменению нужен пакет с PyPI — почти наверняка ему не место в этом проекте; сначала откройте issue для обсуждения.
- **Python 3.12+.** Совпадает с базовым Docker-образом (`python:3.12-slim`) и CI.
- **Оба README — часть контракта.** `README.md` (английский) и `README.ru.md` (русский) — переводы друг друга. Любое пользовательское изменение (CLI-флаги, рецепты `Justfile`, env-переменные, схема конфига, Docker, UI) должно обновлять **оба** файла в одном PR. См. `.pi/skills/readme-sync/SKILL.md`.
- **Никогда не коммитьте runtime/приватные файлы.** `.env`, `dashboard_config.local.json`, `dashboard_overrides.json`, `index.html`, `data.json`, `*.state.json`, `.pids/`, `__pycache__/` — в `.gitignore`, так и должно остаться.

## Локальная настройка

```bash
git clone git@github.com:dream-x/simple_dash.git
cd simple_dash
cp .env.example .env
cp dashboard_config.json dashboard_config.local.json   # сюда — вашу реальную сеть
just validate
just dev                                                # watcher + сервер на http://localhost:8080
```

`Justfile` — канонический task runner (`brew install just`). Он автоматически подгружает `.env`. Те же переменные использует Docker Compose. `just --list` показывает все рецепты.

Итерируйтесь **точечными сканами**, а не полными свипами, чтобы не нагружать роутер:

```bash
just scan-host 192.168.1.152                  # один хост, порты из каталога
PORTS="8000-9000,32400" just scan-host 1.2.3.4
just test                                     # набор тестов на stdlib unittest
```

## Перед открытием PR

Всегда запускайте, и оба должны проходить:

```bash
just validate   # py_compile + json.tool по конфигам и ports_catalog.json
just test       # юнит-тесты (tests/)
```

CI (`.github/workflows/ci.yml`) запускает те же два шага — PR, который их не проходит, не смержится.

Новое поведение в `generate_dashboard.py` должно сопровождаться юнит-тестом в `tests/` (stdlib `unittest`, без доп. зависимостей). Изолируйте тесты — пишите сгенерированные артефакты в `tempfile.TemporaryDirectory()`, а не в рабочую директорию.

## Что куда добавлять

`generate_dashboard.py` — один файл из пяти концептуальных слоёв: загрузка конфига, обнаружение хостов, сканирование портов, fingerprint сервисов и рендеринг. Добавляя что-то, проведите это через все затрагиваемые слои:

| Изменение | Затронуть |
|---|---|
| **Новый CLI-флаг** | argparse в `main()` → `_args` в `Justfile` → `.env.example` → `append_arg` в `docker-entrypoint.sh` → оба README |
| **Новый ключ конфига** | соответствующий `parse_*` / `*_from_dict` → dataclass `DashboardConfig` → `dashboard_json` (если должен попасть в `data.json`) → оба README |
| **Новое определение сервиса** | `detect_app` и/или `DEFAULT_SERVICE_DEFS` / `ports_catalog.json` |

Помните про наследование конфигов: одновременно могут быть активны три файла — `dashboard_config.json` (публичный пример) ← `dashboard_config.local.json` (приватный) ← `dashboard_overrides.json` (снимок). Dataclass `DashboardConfig` — это результат их слияния; не считайте, что ключ из публичного примера и есть актуальное значение.

## Проектные ограничения (соблюдайте, пожалуйста)

- **Никакого полного скана `1-65535` по умолчанию.** Обнаружение использует `ports_catalog.json` плюс опциональные диапазоны `PORTS`. Широкие сканы в больших сетях вешают роутеры.
- **ICMP/ARP — только для обнаружения хостов** — никогда не показывайте их как индикатор здоровья сервиса в UI. Статус сервиса — `online | auth | offline | error | unknown`, выводится из TCP/HTTP-проб.
- **`dashboard_config.json` — безопасный публичный пример.** Реальные сетевые детали — в `dashboard_config.local.json`.
- **Нет авторизации, слушает `0.0.0.0`.** Это инструмент для использования внутри LAN. Не добавляйте функции, предполагающие, что его безопасно выставлять наружу.

## Сообщения об ошибках

Сообщая о баге, укажите: ОС, версию Python, точный рецепт `just` или команду, релевантный `.env`/конфиг (с замаскированными приватными IP/хостнеймами) и что ожидали против того, что произошло.

**Безопасность:** поскольку дашборд без авторизации и раскрывает топологию сети, сообщайте о проблемах безопасности приватно мейнтейнеру, а не в публичном issue.
